# Спека: Minecraft-лаунчер для своего сервера

Дата: 2026-07-04
Статус: утверждена пользователем (vasenka)

## Цель

Windows-лаунчер для частного Minecraft-сервера «для себя и друзей». Друг скачивает
один установщик .exe, входит по нику или через Microsoft, лаунчер автоматически
скачивает сборку модов с GitHub, держит её обновлённой и запускает игру с
автоподключением к серверу.

## Требования

- Платформа: только Windows (один .exe-установщик).
- Авторизация: и офлайн-ники (лицензия есть не у всех), и вход через Microsoft.
- Сборка модов хостится на GitHub (репозиторий + Releases), лаунчер сам её обновляет.
- Красивый фирменный интерфейс: фон, логотип, статус сервера, новости.
- Игроку не нужно ничего ставить вручную (включая Java).
- Сам Minecraft-сервер и наполнение сборки — за рамками этого проекта (сервера
  ещё нет; лаунчер поддерживает и Forge, и Fabric, чтобы выбор сборки не блокировал).

## Архитектура

Electron-приложение:

- **main-процесс (Node.js)** — скачивание файлов, проверка хэшей, запуск игры,
  авторизация. Вся работа с диском и сетью здесь.
- **renderer (HTML/CSS/JS)** — интерфейс; общается с main через IPC.
- **Сборка и дистрибуция** — electron-builder → NSIS-установщик; автообновление
  самого лаунчера через GitHub Releases (electron-updater).

Ключевые библиотеки: `minecraft-launcher-core` (скачивание и запуск игры,
Forge/Fabric), `msmc` (Microsoft OAuth), `electron-updater` (обновление лаунчера).

## Компоненты

### 1. Интерфейс (renderer)

- **Экран входа**: поле ника (офлайн) или кнопка «Войти через Microsoft».
  Выбор запоминается.
- **Главный экран**: кнопка «Играть», прогресс загрузки (полоса + текущий файл),
  статус сервера (онлайн/офлайн, число игроков), блок новостей, кнопка настроек.
- **Настройки**: объём выделяемой памяти, папка установки игры, выход из аккаунта.

### 2. Авторизация

- **Microsoft**: библиотека `msmc`, окно OAuth. Требуется зарегистрировать
  приложение в Azure (бесплатно) и вписать client id в конфиг лаунчера.
- **Офлайн**: игрок вводит ник; UUID генерируется детерминированно из ника
  (стандартная схема `OfflinePlayer:<ник>`, UUID v3), чтобы инвентарь на сервере
  не слетал при переустановке.
- Сессия сохраняется локально, повторный вход не нужен.

### 3. Обновление сборки

- В отдельном публичном GitHub-репозитории лежит сборка и `manifest.json`:
  версия сборки, версия Minecraft, тип загрузчика (Forge/Fabric), адрес сервера,
  список файлов с путями, SHA-1 и URL.
- При каждом запуске лаунчер сверяет локальные файлы с манифестом:
  докачивает недостающие, заменяет изменённые (по хэшу), удаляет из папки
  `mods/` файлы, которых нет в манифесте (конфиги и сейвы игрока не трогаем).
- **Скрипт-упаковщик** (для владельца): кладёшь моды/конфиги в папку, запускаешь
  скрипт — он генерирует манифест и публикует релиз на GitHub.

### 4. Запуск игры

- `minecraft-launcher-core`: качает ванильную версию, ставит Forge/Fabric,
  скачивает подходящую Java (Adoptium JRE) в папку лаунчера.
- Запуск с выбранной памятью и автоподключением к серверу
  (адрес берётся из манифеста).

## Поток данных

запуск лаунчера → проверка обновлений самого лаунчера → вход (или
восстановление сессии) → загрузка манифеста с GitHub → синхронизация файлов
сборки → запуск игры → автоподключение к серверу.

## Обработка ошибок

- **Нет интернета / GitHub недоступен**: играем с последней скачанной версией
  сборки, показываем предупреждение.
- **Битая загрузка**: хэш не сошёлся → повторная попытка (до 3 раз), потом
  понятная ошибка.
- **Игра не запустилась**: окно с хвостом лога и кнопкой «Скопировать», чтобы
  скинуть владельцу.

## Тестирование

- Unit-тесты на логику синхронизации манифеста (сравнение, докачка, удаление
  лишнего, защита конфигов игрока) — единственное действительно хитрое место.
- Ручная e2e-проверка: чистая машина → установка → вход обоими способами →
  скачивание сборки → запуск игры.

## Вне рамок (потом, отдельными проектами)

- Поднятие и настройка Minecraft-сервера (offline-mode + защита от чужих).
- Сборка модпака (выбор Forge/Fabric и модов).
- Регистрация Azure-приложения для Microsoft-входа (5 минут, по инструкции).
